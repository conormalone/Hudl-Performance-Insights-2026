## Code Assessment: Friday's Model 1

### ✅ What's Good
- **Config:** `PressureConfig` dataclass — all tuning params in one place
- **Efficiency:** Uses `scipy.cdist` for pairwise distances — vectorised per frame
- **GK handling:** Dual heuristic (jersey #1 + mean x position near goal line) — reasonable
- **Smoothing:** Savitzky-Golay window=7, polyorder=2 — standard choice, inline with F-0.5.1 decision
- **Transitions:** Works purely from `team_in_possession` flips — no event data needed
- **Composite:** Clear formula: `1 + Σ(indicator_i / baseline_i)`
- **CLI:** `run_pressure.py` has `--match`, `--all`, `--sample-dir` flags

### ⚠️ What Needs Attention
1. **`goalkeeper` column exists in raw data** — we're computing our own heuristic (_resolve_goalkeeper in load_tracking.py + _flag_goalkeepers in opponent_proximity.py), but the parquet already has a `goalkeeper` column. Check if it's reliable.
2. **`output_dir` hardcoded** — `/mnt/project_data/project/focus-fatigue/` — verify this path exists on USB.
3. **Speed values include some outliers** — 94 m/s and 129 m/s max speeds in sample data (likely goalkeeper throw tracking noise). The smoother handles this but worth noting.
4. **`team_in_possession` null ~42%** — expected during ball-out-of-play, but transition detection needs to handle this gracefully (already does).

### Verdict
**Reuse all. Do not rewrite.** Only test, tune, and integrate.

## Shape.json Superpower — Specific Note

The work plan identifies that shape.json provides `averageRolePositionX/Y` per role per 5-minute window. This makes Signal 1 trivially simple:

```python
# Pseudocode for Signal 1 with shape.json
def compute_drift(player_pos, shape_roles, player_id, minute_window):
    role_centroid = shape_roles[player_id]["avg_x"], shape_roles[player_id]["avg_y"]
    drift = euclidean(player_pos, role_centroid)
    return drift  # per-frame, aggregated per-block
```

**No EFPI clustering. No Hungarian assignment. No formation templates.** This saves an estimated 3-5 hours vs the original task plan.

**Prerequisite:** Build the player-to-shape-role bridge (maps tracking integer `player_id` + `jersey_no` + `team_id_opta` → shape.json string UUID role). This is the only non-trivial part.

## Output Schema (Standardised Signal Output)

All signals produce per-block per-player DataFrames with:

| Column | Type | Description |
|--------|------|-------------|
| `game_id` | str | Match identifier |
| `block_id` | str | e.g. "1_3" (phase_block) |
| `player_id` | int | From tracking |
| `team_id_opta` | int | Opta team ID |
| `signal_name` | str | e.g. "transition_latency" |
| `signal_value` | float | Per-block metric |
| `n_frames` | int | Frames in block for this player |
| `phase` | int | 1 or 2 |

Unified dataset built by concatenating all signal outputs and pivoting on `signal_name`.
